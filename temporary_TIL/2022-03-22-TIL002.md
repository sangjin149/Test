## :star: 온라인 강의

### 1.네트워크 기초(aka.검색창에 구글을 입력한다면?)

검색창에 주소를 입력하면 다음과 같은 절차를 걸쳐 브라우저에 화면이 나타난다.

1. URL 파싱
2. 수신자 IP 서버 주소 조회
3. 수신자 IP가 존재하는 서버에 요청 전달
4. 서버가 ARP를 사용하여 해당 IP를 가진 클라이언트에게 MAC주소 수신
5. 반환받은 MAC주소를 가진 수신자와 통신경로를 열기 위해 Socket개방 요청
6. 통신이 시작되면 수신자는 요청에 따라 html파일 등의 응답 반환
7. 브라우저가 6에서 받은 파일을 렌더링

### 2.컴퓨터 시간의 원리

1.시스템 클럭: 메인보드에 붙어있는 리얼타임클럭(RTC)라는 부품이 카운터 회로를 통해 클럭을 발생시킨다.
카운터 회로에는 결정 진동자라는 부품이 1클럭에 32.768khz를 발생시킨다.

2.NTP를 이용하여 현재 시간을 알애내기: 계층 구조로 이루어진 NTP 서버에 네트워크 요청을 하여 현재 시간을 받아낸다. 구조 최상단에는 원자시계에 직접 연결된 노드가 있다.

### 3.함수형 프로그래밍

함수형 프로그래밍의 특징:

1. 추상화의 최소 단위가 함수다.
2. 함수 단위로 나누어져 재사용성이 높다.
3. 불변성을 지향하기에 동작을 예측하기 쉽고 사이드 이펙트를 방지한다. 동시성 문제도 동시에 해결 가능하다.

함수형 프로그래밍의 장단점:

1. 상태가 없기 때문에 사이드 이펙트가 없다 => 여러 상태가 존재해야할 때 많은 리소스를 써야한다.
2. 재사용성이 높다 => 함수를 기능별로 많이 쪼개야한다.
3. 코드가 짧고 간결하다 => 높은 숙련도를 필요로한다.

:dizzy: 불변성을 지향하기 때문에 사이드 이펙트를 방지한다는게 어떤 뜻인지 모르겠다. 다음주에 함수형 프로그래밍을 공부할 때 더 알아봐야겠다.

### 5.이벤트 루프

JS 가상 머신은 한 개의 스레드를 가지지만 웹브라우저가 멀티스레드를 지원하기 때문에 동시에 여러 일을 처리할 수 있다. 비동기 함수는 Web API의 스택에 차례대로 저장되었다가 JS 머신의 콜스택이 다 비워지면 Task Queue에 한꺼번에 옮겨진다. 이후 Call Stack이 비어있으면 Task Queue의 맨 앞에 있는 함수가 Call Stack으로 옮겨져 실행된다.

### 7.유니코드

* CCS(Coded Character Set): 문자와 정수화된 식별자 쌍의 집합
* CES(Character Encoding Scheme): CCS의 정수화된 식별자를 8비트 집합에 대응시킨 것
* TES(Transfer Encoding Syntax): 인코딩한 문자가 특정 프로토콜을 타고 전송되도록 변환하는 것
